git reset HEAD 的作用不是unstage吗？为什么运行后原本期望删掉的test02.md又还原到未删除之的commit状态？

我的理解是，git reset HEAD 的确做了unstage，但并不仅仅只做unstage。对staged状态进行回退，指的是往上一次commit的方向回退。而且，这里的git reset HEAD 和之前回退add时候的命令虽然名字相同，但是存在本质区别，这里的更重量级，reflog可以显示reset记录。

《Pro Git》中强调过，任何提交到Git的数据都可以被恢复，但没提交过的数据，对Git来说就像从未存在过一样。

可以通过git reset --hard xxxx指定到任何commit的状态，适用git log可以在控制台打印出最初到当前提交的所有信息，但是问题来了。
